
1.CPU 성능 검증: "압도적인 병목 지점(Hotspot) 확인
"데이터 분석:사진을 보면 get_gcd 함수가 전체 CPU 사용량의 **99.14%**를 차지하고 있습니다.
검증 결과: 이는 프로그램의 부하가 오직 GCD 계산에만 집중되어 있음을 보여줍니다.
피보나치 수열을 이용한 최악의 경우(Worst Case) 테스트가 수행되었으며, 다른 부수적인 연산(출력, 변수 할당 등)이 알고리즘 성능 측정에 방해가 되지 않았음을 증명합니다.
2. 실행 효율성 검증: "셀프 CPU 시간의 의미"데이터 분석: get_gcd 함수의 **셀프 CPU(단위, %)**가 **95.87%**로 나타납니다.
검증 결과: 함수 내부의 로직(반복문과 나머지 연산)이 매우 직접적이고 군더더기 없이 실행됩니다. 
불필요한 함수 호출이나 메모리 접근 없이 순수하게 유클리드 호제법의 논리만으로 빠르게 연산이 처리되고 있습니다.
3. 공간 복잡도(Memory) 검증: "안정적인 자원 관리"
데이터 분석: 첫 번째 사진의 메모리 그래프가 입력값 $F(n)$의 증가에도 불구하고 **완벽한 수평선(Flat Line)**을 유지합니다.
검증 결과: 입력 숫자가 커지더라도 추가적인 메모리(Stack 또는 Heap)를 전혀 사용하지 않는 $O(1)$ 공간 복잡도를 실증합니다. 
재귀(Recursion) 대신 반복문(Iteration)을 사용하여 스택 오버플로우 위험 없이 안전하고 효율적으로 설계되었음을 보여줍니다.
